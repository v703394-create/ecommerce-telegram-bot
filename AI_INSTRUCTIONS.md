AI INSTRUCTIONS (Russian Interaction Mode)

Основное правило общения

Пользователь общается на русском языке.

AI должен:

отвечать на русском

использовать простые и понятные объяснения

избегать сложной академической терминологии

давать практические инструкции


Если пользователь пишет на русском — отвечать только на русском.


---

Стиль ответов

Ответы должны быть:

1️⃣ короткими
2️⃣ пошаговыми
3️⃣ без лишнего текста

Предпочтительный формат:

Шаг 1
...

Шаг 2
...

Не выдавать длинные инструкции сразу.

Давать один шаг за раз, если пользователь так просит.


---

Формат кода

Если пользователь просит код:

давать полный файл

не давать фрагменты

не писать "найди строку и добавь"


Формат:

Открой файл:

shop/models.py

Замени содержимое на:

после этого полный код файла.


---

Контекст проекта

Этот репозиторий содержит код проекта:

Kraken Shop

Это Telegram Mini App магазин.

Backend написан на:

Python
Django
SQLite

Основная цель:

создать простой магазин для Telegram Mini App.


---

Ограничения проекта

Проект намеренно небольшой.

Ожидаемая нагрузка:

50–100 пользователей
2–3 товара

Поэтому система должна быть:

простой

стабильной

без лишней сложности



---

Архитектура системы

Основная архитектура:

Telegram Bot
      │
      ▼
Telegram Mini App (WebApp)
      │
      ▼
Django Backend API
      │
      ▼
Database


---

Роль Telegram бота

Бот выполняет несколько задач:

1️⃣ точка входа

/start

2️⃣ кнопка открытия Mini App

Открыть магазин

3️⃣ уведомления администратору

4️⃣ подтверждение заказов


---

Роль Mini App

Mini App — это витрина магазина.

Он показывает:

список товаров

карточки товаров

фото

цену

кнопку "Купить"


Mini App обращается к backend API.


---

Backend задачи

Django backend отвечает за:

хранение товаров

хранение заказов

API

админ панель

загрузку изображений



---

Основные модели

Проект использует три основные модели.

Product

Товар.

Поля:

name
description
price
photo1
photo2
is_active
created_at


---

Order

Заказ пользователя.

Поля:

channel
external_user_id
product
status
created_at

Статусы:

PENDING
APPROVED
DECLINED


---

Review

Отзыв на товар.

Поля:

product
external_user_id
rating
text
created_at


---

Логика покупки

1️⃣ пользователь открывает Mini App

2️⃣ Mini App загружает товары

GET /api/products

3️⃣ пользователь нажимает Купить

4️⃣ Mini App отправляет запрос

POST /api/orders/create

5️⃣ создаётся заказ

status = PENDING

6️⃣ администратор получает уведомление

7️⃣ администратор подтверждает или отклоняет


---

Платформенная независимость

Система должна работать не только с Telegram.

В будущем планируется поддержка:

Signal
WhatsApp
Web сайт

Поэтому используется поле:

external_user_id

вместо Telegram-специфичных идентификаторов.


---

Правила разработки

AI должен:

1️⃣ избегать сложной архитектуры
2️⃣ избегать микросервисов
3️⃣ не добавлять Redis / Celery без необходимости
4️⃣ сохранять простой Django backend
